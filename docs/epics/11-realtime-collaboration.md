# Epic 11 — Real-time collaboration & presence

**Phase:** 3 — Business surface

## Goal

A live workspace where members see each other's presence and activity as it happens.

## Why it matters

"Peers" must feel real — agents appear as working/online alongside humans.

## Key stories

- WebSocket presence and a unified activity feed (agent and human actions look identical).
- Live board, artifact, and task updates.
- Agent status ("working", "waiting for approval", "idle") surfaced like human presence.

## Acceptance criteria

- Board and artifact changes appear live.
- Agents show accurate working status.
- The activity feed treats both member types uniformly.
